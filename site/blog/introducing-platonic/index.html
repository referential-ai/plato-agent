Introducing Platonic: visible work, replayable by default

A framework for disciplined agent execution, made concrete by Plato Agent.

July 15, 2026 Referential AI

Agent systems become difficult to trust when their most important choices are hidden inside control flow. Platonic begins from a stricter premise: work that changes an outcome should cross a typed boundary and leave durable evidence.

The framework

Platonic is the common foundation. It assembles bounded context, separates tool proposals from policy decisions, and treats the event ledger as the source of truth for replay and audit. Hosts still own real IO; the framework keeps the execution contract small and inspectable.

The reference runtime

Plato Agent is the named application and reference runtime built on Platonic. It connects providers, local tools, approvals, persistence, daemon clients, and the desktop experience without moving those concerns into the kernel.

A session can continue across clients, pause for a local approval, recover after interruption, and replay from recorded events. The application shows the framework under real product pressure while preserving the boundary between durable semantics and host mechanics.

Start with evidence

The quickest way to understand the system is to run a bounded task and replay its ledger. The canonical quickstart covers the build, first run, approvals, and offline readback without creating a second documentation path here.
